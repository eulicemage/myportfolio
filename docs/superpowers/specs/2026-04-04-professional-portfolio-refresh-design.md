# Professional Portfolio Refresh Design Spec

Date: 2026-04-04
Project: `myportfolio`
Topic: Modern professional portfolio refresh for software developer positioning

## Goal

Refresh the current portfolio so it presents the owner more professionally as a software developer while keeping the site modern, attractive, and easy for employers to scan.

The redesign should:

- Present the owner clearly as a software developer
- Feel modern and polished rather than flashy or template-like
- Replace self-rated skill bars with proof-based technical presentation
- Add `EBPLS` as a featured project with LGU-oriented context
- Keep other projects as supporting portfolio work
- Improve the contact section so employers can act quickly
- Preserve responsiveness and accessibility basics

## Audience

Primary audience:

- Recruiters
- Hiring managers
- Technical interviewers
- Employers reviewing junior software developer candidates

Secondary audience:

- Potential collaborators
- Internship and freelance opportunities

## Positioning

The refreshed portfolio should position the owner as:

- A software developer with modern web development capability
- A fresh graduate with real project experience
- Someone who can build usable interfaces and full-stack solutions
- A candidate who is professional, capable, and ready to grow

The site should avoid exaggerated claims and instead let projects, stack, and clarity carry credibility.

## Creative Direction

Selected direction: `Modern Product Portfolio`

### Visual principles

- Clean, structured, product-style layout
- More mature visual system than the current neon-heavy look
- Strong typography hierarchy
- Subtle motion instead of constant high-energy animation
- Better spacing, cleaner cards, and clearer content flow

### Tone

- Modern
- Professional
- Confident
- Clear
- Employer-friendly

## Information Architecture

The page should be organized into:

1. Hero
2. About / value summary
3. Core stack and technical strengths
4. Featured projects
5. Contact / hiring CTA

## Section Design

## 1. Hero

Purpose:

- Establish role identity immediately
- Create a professional first impression
- Guide employers to the most important actions

Content direction:

- Name displayed clearly
- Main role identity centered on `Software Developer`
- Short supporting statement about building modern web experiences
- Primary CTA to view projects
- Secondary CTA to contact

Design direction:

- More mature color balance
- Cleaner layout and spacing
- Keep motion subtle and supportive
- Typing effect may remain, but it should feel restrained

## 2. About / Value Summary

Purpose:

- Give hiring context without sounding inflated

Content direction:

- Short summary of being a fresh graduate software developer
- Emphasis on building real projects and continuing to grow
- Focus on value to a team rather than generic self-description

This section should be concise and easy to scan.

## 3. Core Stack and Technical Strengths

Purpose:

- Replace arbitrary percentage bars with credible technical presentation

Replace the current skill meter approach with grouped technical strengths such as:

- Frontend: React, Next.js, JavaScript, Tailwind CSS, responsive UI
- Backend: Node.js, Express, REST APIs
- Data: Supabase, Firebase, MySQL
- Workflow: GitHub, Figma, problem solving, collaboration

Presentation style:

- Clean grouped cards or chips
- Practical wording
- Focus on tools actually used in projects

## 4. Featured Projects

Purpose:

- Make project work the main proof of capability

Project structure:

- `EBPLS` should be introduced as a featured project
- It should be described as an LGU-oriented system
- It should explicitly mention React, Next.js, Node/Express, and Supabase
- The project summary should emphasize workflow relevance and real-world use context

Supporting projects:

- LU BAO Merchandise
- Rockies Fitness
- Floral Haven

These supporting projects should remain in the portfolio, but visually and structurally they should support the featured project rather than compete equally with it.

## 5. Contact / Hiring CTA

Purpose:

- Help employers contact the owner quickly and confidently

Content direction:

- Strong closing statement about being open to software developer opportunities
- Email, LinkedIn, GitHub, and location remain visible
- Contact form can remain, but the layout should feel cleaner and more professional
- Direct contact paths should be obvious

## UX Requirements

- Improve hierarchy and readability
- Reduce visual noise
- Preserve section-based navigation
- Improve mobile spacing and readability
- Keep interactions smooth but not distracting
- Make the portfolio easier to scan quickly

## Accessibility and Quality

The refresh should preserve or improve:

- semantic structure
- keyboard-accessible navigation
- readable contrast
- clear headings
- mobile responsiveness
- reduced visual clutter

## Implementation Scope

In scope:

- Refine hero layout and copy
- Replace current skill percentage section
- Add `EBPLS` as featured project content
- Improve project hierarchy
- Polish contact section and CTA
- Tune colors, spacing, and animation for a more professional look

Out of scope:

- Multi-page site conversion
- Backend contact integration beyond current behavior
- Large architectural refactors outside the portfolio component and styling

## Risks and Guardrails

Risks:

- Over-design can reduce clarity
- Over-claiming can weaken trust
- Too much animation can feel less professional

Guardrails:

- Use modern polish, not excessive effects
- Keep content truthful and grounded
- Make software developer identity clear
- Use projects as the main evidence of skill

## Success Criteria

The refresh succeeds if:

- The portfolio looks more professional within the first few seconds
- Employers can understand the owner’s role and strengths quickly
- `EBPLS` stands out as a meaningful featured project
- The site feels more modern and mature than the current version
- The portfolio remains readable and polished on mobile and desktop
